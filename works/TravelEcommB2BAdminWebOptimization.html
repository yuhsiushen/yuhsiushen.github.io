Online Booking Platform

Project brief

This product enables providers to list, edit accommodations, manage inventory, pricing, and orders via a Web platform.
With the overhaul of the 2C product, adjustments are needed for 2B functionalities. Yet, due to outdated programming, a foundational renovation is necessary, presenting an opportunity to address long-standing interface and user experience issues.

Main challenges

Lacks of user data & feedback

Early website development lacked tracking, resulting in a reliance on indirect feedback from business units rather than data-driven insights and direct user feedback.

Hard to focus on main pain points

Without backstage data feedback and a fixed iteration frequency, the team has limited visibility into product pain points, hindering their ability to prioritize critical issues.

Outdated codebase & lengthy development

Early website use of outdated codebase necessitated more time for subsequent engineers to familiarize themselves with the system, prolonging development timelines and slowing down development speed.

Understand the problem

User interview and usability testing

We categorized users into 4 types based on the platform accommodation types: hotels, management consultants, BnBs, and campgrounds. We invited 2-3 users from each category for interviews, totaling 11 interviews.
Due to the pandemic, interviews were conducted remotely. Mainly focused on usability testing for functions like room availability, pricing, promotions, and accommodation information management, as well as discussing other product issues encountered by users.

Pain point 1:
Difficulties in slow website speed

Outdated website structure causes delays when editing accommodation details, leading to slow loading times and occasional timeouts, making it less user-friendly.

Pain point 2:
Inconvenient inventory and pricing settings

Previous inventory pricing methods didn't match user habits, requiring users to spend more time selecting dates instead of using options based on weekdays, weekends, or consecutive holidays.

Pain point 3:
Limited flexibility in discount setup

Before the redesign, promotions could only be set with start and end dates and fixed discount percentages, lacking options to differentiate discounts for different days or specify discount periods, reducing flexibility.

User journey and opportunities

Opportunity 1:
Information setup exp. enhancement

Accommodation information setup serves as the first touchpoint for property owners on our platform. Optimizing the setup and editing experience not only helps owners complete tasks faster but also reduces confusion for travelers when viewing accommodation details.

Opportunity 2:
Operational features enhancement

Pricing, inventory, and discount settings are the most commonly used functions for operators.
Optimizing operational functionalities assists property owners in managing accommodations more smoothly.

Opportunity 3:
Order management features enhancement

Managing orders is a frequent task for property owners. Convenient order management functionalities increase efficiency in order confirmation and reduce error rates.

Componentization & RWD

Previously, our outdated program structure led to slow webpage loading and user inconvenience.
During this refactoring, we're redesigning webpage breakpoints and components to enhance speed and development efficiency.

Optimized address setup UX

Many guests contact customer service to inquire about accommodation addresses, and providers often need platform assistance to establish correct addresses due to system restrictions.
Upon investigation, it was found that the previous address setup method was ineffective at verifying accurate accommodation positioning.
Therefore, during the redesign, the process was optimized to help providers confirm correct locations.
Additionally, accommodations not locatable on Google Maps were reminded to provide directions for guests.

Optimized photo upload UX

Users wanted more detailed photos, and accommodations needed to upload more for better promotion. System limitations restricted this before.
In the redesign, we lifted the photo quantity limit and streamlined the upload process, making it easier for accommodations to upload multiple photos at once and reducing user confusion.

Adjusted date picker

User interviews showed that users often need to set inventory and prices based on different weekdays, distinguishing regular from peak periods.
Previously, users could only select date ranges using a calendar, requiring multiple operations for weekly settings. So, we simplified the process in the redesign to save users time and steps.

Added continuous holidays exclusion feature

In the context of users setting prices and inventory, there's often a need to exclude continuous holidays.
Previously, users had to manually exclude them. This adjustment adds a feature to exclude continuous holidays, reducing user setup time.

Added limit consecutive nights for stay

Before redesign, early bird promotions could only be set a few days in advance with discounts.
This adjustment now allows users to set limitations on consecutive nights for stay, making it easier for them to configure.

Optimized discount applicable dates setting

  1. Added date range setting:
    Users can now set specific start and end dates for check-in, with the flexibility to customize based on the day of the week.This allows for tailored discounts according to different periods like weekdays, peak seasons, or consecutive holidays.
  2. Enhanced ordering period setting:
    Previously, discounts took effect immediately upon submission. In this update, users can now set the ordering period during which the discount applies, offering greater flexibility for setting up flash sales, holiday promotions, and more.

Optimized discount exclusion dates

Before the update, excluding dates from promotions required selecting them individually on a calendar. If multiple months shared the same excluded weekdays, users had to repeat the process multiple times, which was time-consuming and laborious.

The optimization allows users to exclude specific weekdays or dates when setting up promotions for meeting user needs.

Room for improvement

Order management enhancement

Currently, the order management function only allows viewing order details, lacking features like exporting details or responding to guest reviews. Expanding this function would streamline property management.

Display discounted room rates

Currently, room rates and promotions are shown separately, requiring property owners or hotel operators to calculate final rates manually. Adding a feature to display final rates would simplify operations.

Bulk operations for room inventory & pricing

While recent enhancements help property owners adjust availability and pricing for different periods, it remains time-consuming for properties with many rooms. Implementing bulk operations would simplify this process.

Provide performance date

The current backend lacks data on accommodation operational performance. If such data were available, property owners could easily track the performance of different business operations from the backend, thereby increasing its usage.

Continuous establish component library

This optimization initiates the integration of the backend component library. Further pages in the backend require componentization to ensure consistent styles and interactions, enhancing user clarity and website stability.
